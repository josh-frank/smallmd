# Contributing

smallmd is intentionally minimal. Contributions that add complexity without clear necessity will be closed.

---

## Bug fixes

Open an issue before submitting a fix. One bug per PR. Include a reproduction case.

## Features

Open an issue first. New features must not introduce dependencies and should not meaningfully increase the codebase size. If in doubt, it's a fork.

## Themes

Add a folder under `themes/`. Must work without JavaScript. PR should include a screenshot.

## Docs

No issue needed. Fix and open a PR.

---

## Rules

- No new dependencies
- No build steps
- PRs should be small and focused
- Existing behavior should not change unless fixing a bug
